# Releases

Changelog for the **Worldwide Events** TradingView indicator.

Each file contains the release note for one version — a concise summary of what
changed in that release. These notes are meant to be pasted into TradingView's
**release notes** field when publishing an update.

> **Note:** A release note is a _changelog_ — only what changed in this version.
> It is **not** the full indicator description. TradingView shows the original
> description once, at the top of the script page; release notes are appended
> below it, in order. Repeating the full description in every release note makes
> the script page look cluttered.
>
> The indicator **description** (the evergreen "what it is" text) lives in
> [../docs/DESCRIPTION.md](../docs/DESCRIPTION.md), not here.

## Versioning

`MAJOR.MINOR.PATCH`

- **PATCH** — new events added, descriptions/links fixed, no behavior change.
- **MINOR** — new indicator features (new language, new setting, new visuals).
- **MAJOR** — breaking changes to inputs or output.

The event count is the number of dated entries in `data/events-en.md`.
